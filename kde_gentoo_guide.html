KDE Plasma 6 on Gentoo Linux

Full plasma-meta + kde-apps-meta installation guide for a completed Gentoo base system. Assumes a working kernel (EFI stub), AMD GPU, CPU flags, and desktop/plasma profile already configured. Based on the Gentoo wiki pages for KDE, SDDM, elogind, PipeWire, and NetworkManager.

OpenRC amdgpu / radeonsi EFI Stub Plasma 6 Wayland elogind
Source: All commands and notes on this page are derived from wiki.gentoo.org/wiki/KDE, SDDM, Elogind, PipeWire, and NetworkManager wiki pages. Cross-reference those pages — they are updated more frequently than any guide.

00 Base System Prerequisites

This guide assumes you have a fully working Gentoo base system before proceeding. The following must already be in place:

ComponentExpected State
KernelCompiled, installed, and booting via EFI stub (no GRUB)
VIDEO_CARDS*/* VIDEO_CARDS: -* amdgpu radeonsi in /etc/portage/package.use/00video_cards
CPU_FLAGS_X86Set via cpuid2cpuflags in /etc/portage/package.use/00cpu-flags
Profiledefault/linux/amd64/23.0/desktop/plasma selected
Init systemOpenRC (not systemd)
BootSystem boots to a working TTY login
linux-firmwareInstalled (required for AMD GPU firmware blobs)

Quick verification:

# Profile is desktop/plasma (OpenRC)?
eselect profile show
# Expected: default/linux/amd64/23.0/desktop/plasma

# VIDEO_CARDS set?
cat /etc/portage/package.use/00video_cards
# Expected: */* VIDEO_CARDS: -* amdgpu radeonsi

# CPU flags set?
cat /etc/portage/package.use/00cpu-flags
# Expected: */* CPU_FLAGS_X86: aes avx avx2 ...

# Booting via EFI stub?
efibootmgr -v

# Kernel running?
uname -r
Warning

Use desktop/plasma — not desktop/plasma/systemd. This entire guide assumes OpenRC. If your profile is wrong, set it with eselect profile set default/linux/amd64/23.0/desktop/plasma before continuing.

01 Add KDE USE Flags

VIDEO_CARDS and CPU_FLAGS_X86 are already handled in package.use. The plasma profile manages most KDE-specific USE flags automatically. You only need to add session management, audio, and desktop USE flags to make.conf. Remove any existing qt4, kde, plasma, or handbook entries — the profile manages these now.

Append these USE flags to your existing make.conf:

# /etc/portage/make.conf — ADD to existing file

# KDE / desktop USE flags (append to existing USE or add new USE line)
USE="elogind dbus policykit pipewire alsa bluetooth networkmanager \
     wayland X openrc -systemd -gnome -gtk-doc -test"

ACCEPT_LICENSE="*"
ACCEPT_KEYWORDS="amd64"
Critical

The Gentoo elogind wiki explicitly states: using elogind and systemd USE flags simultaneously is untested and not advised. Keep -systemd in USE at all times.

Check for leftover conflicting flags:

grep -e "qt[0-9]" -e "kde" -e "plasma" -e "handbook" /etc/portage/make.conf

02 Pre-install Services

The Gentoo KDE wiki states that default service dependencies are pulled in automatically by the plasma profile, but must be set up properly before starting Plasma for the first time. Install them first with --oneshot so Portage accounts for them when resolving KDE's dependency tree:

# Session management — must exist before plasma-meta build
emerge --ask --oneshot sys-auth/elogind

# Message bus
emerge --ask --oneshot sys-apps/dbus

# Audio + screen sharing (Wayland)
emerge --ask --oneshot media-video/pipewire media-video/wireplumber
Note

--oneshot installs without adding to @world but Portage still resolves against them during the main KDE emerge. This avoids circular dependency issues.

03 Install KDE Plasma + Apps

SDDM is pulled in automatically by plasma-meta — no need to install it separately. This emerge takes a while.

emerge --ask kde-plasma/plasma-meta kde-apps/kde-apps-meta
Warning

The Gentoo KDE wiki warns against using kde-plasma/plasma-desktop as a base. It excludes critical packages like powerdevil (power management) and systemsettings. Use plasma-meta. If you hit a circular dependency error (rare since late 2024): USE="minimal" emerge --ask kde-plasma/plasma-meta, then re-run the full emerge.

After plasma-meta, install remaining system tools:

emerge --ask \
    net-misc/networkmanager \
    net-wireless/bluez \
    sys-fs/udisks \
    sys-power/upower \
    app-misc/colord \
    x11-misc/xdg-user-dirs \
    app-admin/sudo \
    gui-libs/display-manager-init

04 Configure display-manager-init

gui-libs/display-manager-init provides the OpenRC init script that SDDM needs. Set it to use SDDM:

sed -i 's/^DISPLAYMANAGER=.*/DISPLAYMANAGER="sddm"/' /etc/conf.d/display-manager

# Verify:
grep DISPLAYMANAGER /etc/conf.d/display-manager

05 SDDM Configuration

SDDM uses directory-based config. Package defaults live in /usr/share/sddm/sddm.conf.d/ — local overrides go in /etc/sddm.conf.d/. The Gentoo SDDM wiki specifies that Wayland mode on OpenRC/elogind requires using KWin as the compositor with specific flags:

mkdir -p /etc/sddm.conf.d/

cat > /etc/sddm.conf.d/override.conf << 'EOF'
[General]
DisplayServer=wayland
GreeterEnvironment=QT_WAYLAND_SHELL_INTEGRATION=layer-shell

[Wayland]
CompositorCommand=kwin_wayland --drm --no-lockscreen --no-global-shortcuts --locale1
SessionDir=/usr/share/wayland-sessions

[Users]
RememberLastUser=true
RememberLastSession=true
EOF
Warning

The Gentoo SDDM wiki states the --drm flag is required in CompositorCommand for DRM-based systems (which AMD is). Without it SDDM may fail to start.

Verify machine-id exists — SDDM requires it:

if [ ! -f /etc/machine-id ]; then
    systemd-machine-id-setup
fi
cat /etc/machine-id   # must be non-empty

06 OpenRC Services

On OpenRC every service must be explicitly enabled. On systemd they auto-start via D-Bus activation — on OpenRC they do not. KDE Plasma requires these services in dependency order:

# elogind — BOOT runlevel, must be up before display manager
rc-update add elogind boot

# D-Bus — everything depends on this
rc-update add dbus default

# NetworkManager — pulled in by plasma-meta
rc-update add NetworkManager default

# Bluetooth
rc-update add bluetooth default

# Disk/removable devices — Dolphin, device notifier
rc-update add udisks default

# Power management — battery, suspend, hibernate
rc-update add upower default

# Colour management
rc-update add colord default

# Display manager (SDDM via display-manager-init)
rc-update add display-manager default
Warning — elogind must be in boot, not default

The Gentoo SDDM wiki states: "In its default configuration, SDDM will not start without elogind. Enable and start it — add elogind to the boot runlevel." This ensures it is running before the display manager initialises.

Verify the full list:

rc-update show | grep -E 'elogind|dbus|Network|bluetooth|udisks|upower|colord|display'

07 Remove netifrc / dhcpcd Conflicts

If you previously used netifrc scripts or dhcpcd, they will conflict with NetworkManager — two DHCP clients fighting each other. The Gentoo NetworkManager wiki gives this exact command:

for x in /etc/runlevels/default/net.* ; do
    rc-update del $(basename $x) default
    rc-service --ifstarted $(basename $x) stop
done
Critical

Remove netifrc entries before rebooting. Two simultaneous DHCP clients will cause unpredictable network behaviour.

08 User Setup

The Gentoo NetworkManager wiki specifies that plugdev is required for non-root users to manage network connections. The seat group is needed for elogind seat management:

# Replace 'youruser' with your actual username
usermod -aG wheel,audio,video,usb,plugdev,input,bluetooth,seat youruser

# Enable wheel group for sudo
visudo
# Uncomment: %wheel ALL=(ALL:ALL) ALL

# Verify:
id youruser

09 Polkit Wheel Rule

By default in Gentoo, only root is an administrator for KDE system dialogs (printers, adding users, etc.). The Gentoo KDE wiki documents that NetworkManager already ships a polkit rule for plugdev, but other dialogs need a wheel rule:

cp /usr/share/polkit-1/rules.d/50-default.rules \
   /etc/polkit-1/rules.d/49-wheel.rules

# Edit 49-wheel.rules, change the group to wheel:
# polkit.addAdminRule(function(action, subject) {
#     return ["unix-group:wheel"];
# });
Note

NetworkManager connection management already works via the plugdev group polkit rule shipped with the Gentoo package. The wheel rule is for other system dialogs like printer configuration.

10 PipeWire

PipeWire on OpenRC uses gentoo-pipewire-launcher — a script Gentoo installs specifically for non-systemd systems. The Gentoo PipeWire wiki states that Plasma reads XDG autostart files, so no manual configuration is needed for a Plasma session:

# PipeWire installs /etc/xdg/autostart/pipewire.desktop automatically.
# Plasma reads this on login and starts PipeWire + WirePlumber.
# No manual action required for a Plasma/Wayland session.

# After first login, verify:
wpctl status
# Should list sinks/sources under your audio hardware
Note

The Gentoo PipeWire wiki notes gentoo-pipewire-launcher is a temporary OpenRC solution while OpenRC user services mature. As of 2026, the XDG autostart method is the correct approach for Plasma sessions — the .desktop file is installed by the package automatically.

11 Plasma Session Control

From Plasma 6.2.1 on Gentoo, kde-plasma/plasma-login-sessions controls which session types appear in SDDM. The desktop/plasma profile enables X globally, so both Wayland and X11 entries appear by default. To restrict to Wayland only:

# /etc/portage/package.use/plasma-login-sessions
kde-plasma/plasma-login-sessions wayland -X

emerge --ask --changed-use kde-plasma/plasma-login-sessions
Warning

Only restrict to Wayland after confirming it works. Test first from a TTY: dbus-run-session startplasma-wayland. A broken Wayland config with X11 disabled will give you a black screen with no fallback.

12 AMD GPU Verification

With amdgpu radeonsi in VIDEO_CARDS and linux-firmware installed, AMD works out of the box. The Gentoo KDE wiki notes KWin falls back to software rendering if VIDEO_CARDS is wrong or DRM is not enabled in the kernel. Verify after first boot:

# Firmware loaded OK?
dmesg | grep -i amdgpu | head -20

# Mesa using AMD hardware?
glxinfo | grep 'OpenGL renderer'
# Expected: AMD Radeon RX... (RADV ...)

# Vulkan present?
vulkaninfo --summary 2>/dev/null | grep deviceName

# Wayland session active?
echo $WAYLAND_DISPLAY    # wayland-0 or wayland-1
echo $XDG_SESSION_TYPE   # wayland

13 KWallet PAM Integration

To have KWallet unlock automatically at SDDM login, configure PAM. The Gentoo KDE wiki documents this for SDDM:

# Append to /etc/pam.d/sddm:
-auth     optional  pam_kwallet5.so
-session  optional  pam_kwallet5.so auto_start
Note

If you do not want KWallet, disable it in System Settings → KWallet, or set a blank password on first launch to suppress prompts entirely.

14 Pre-boot Checklist

# Services correct?
rc-update show

# machine-id present?
cat /etc/machine-id

# User in correct groups?
id youruser

# SDDM config valid?
cat /etc/sddm.conf.d/override.conf

# display-manager set to sddm?
grep DISPLAYMANAGER /etc/conf.d/display-manager

# No conflicting net.* scripts?
ls /etc/runlevels/default/net.* 2>/dev/null || echo "Clean"

# Reboot
reboot

Troubleshooting

ProblemFix
SDDM black screen / no greeter TTY2 → dmesg | grep amdgpu. Check --drm in CompositorCommand. Verify elogind is in boot runlevel not default.
elogind/systemd block conflict Ensure USE has elogind -systemd. Run emerge --ask --changed-use --deep @world.
No audio after login Run wpctl status. If empty, check /etc/xdg/autostart/pipewire.desktop exists. Log out and back in.
KDE system dialogs fail auth Add polkit wheel rule (Section 10). NetworkManager uses plugdev separately.
Drives not appearing in Dolphin rc-service udisks start then rc-update add udisks default. User must be in plugdev.
Suspend/hibernate missing from menu elogind must be in boot runlevel. Check kernel has CONFIG_SUSPEND=y.
NetworkManager not managing wifi Remove netifrc net.* scripts from default runlevel (Section 8).
KWallet prompt loop after login Configure pam_kwallet5.so in /etc/pam.d/sddm (Section 14), or set blank wallet password.
Both X11 and Wayland in SDDM Normal — profile enables X globally. Restrict with plasma-login-sessions USE flags (Section 12).
KWin using software rendering Verify */* VIDEO_CARDS: -* amdgpu radeonsi in /etc/portage/package.use/00video_cards and DRM enabled in kernel. Rebuild Mesa.
