// gentoo linux — kernel setup

CachyOS Kernel
on Gentoo

installkernel + dracut + EFI stub · fresh install · no dist-kernel

~amd64 dracut efistub CachyOS-kernels overlay
// prerequisites
i
Required USE flags for installkernel

Before starting, confirm installkernel has the correct USE flags. Add to your package.use:

/etc/portage/package.use/installkernel
sys-kernel/installkernel dracut efistub

Then reinstall to apply:

bash 
emerge --ask sys-kernel/installkernel
!
Create the EFI directory on your ESP — do this before emerging the kernel

installkernel will not create the EFI subdirectory for you. If it does not exist when the kernel is installed, the postinst phase will fail silently with "Kernel install failed" and no obvious reason in the top-level output.

Make sure /boot is mounted first, then create the folder:

bash 
mount /boot          # skip if already mounted
mkdir -p /boot/EFI/Gentoo
ℹ The subdirectory name (Gentoo) is arbitrary — rEFInd auto-scans all subdirectories under /boot/EFI/ for .efi files, so any name works.
!
Set the kernel cmdline before emerging — required when installing from a chroot

When running inside a chroot, installkernel detects this and refuses to continue unless a kernel command line is explicitly configured. Without it, dracut would fall back to /proc/cmdline of the live ISO — wrong kernel arguments that would cause boot failure.

Find your root partition UUID:

bash 
blkid   # note the UUID of your root partition (not the ESP)

Then create the dracut config file. Note the leading and trailing spaces inside the quotes — the Gentoo handbook requires them:

/etc/dracut.conf.d/00-installkernel.conf 
kernel_cmdline=" root=UUID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx rw "
⚠ Replace xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx with the UUID of your root filesystem partition, not the ESP. Use blkid to find it.
ℹ An alternative location is /etc/cmdline.d/root.conf — both are accepted by installkernel. The dracut config approach is what the Gentoo handbook documents.
// overlay setup
1
Install eselect-repository and git
bash 
emerge --ask app-eselect/eselect-repository dev-vcs/git
2
Add the CachyOS-kernels overlay

Use the Szowisz/CachyOS-kernels overlay — this is the correct one for kernel ebuilds.

bash 
eselect repository enable CachyOS-kernels
emaint sync -r CachyOS-kernels
⚠ If eselect repository enable fails with "not in remote list", manually create the repos.conf entry — see the fallback section below.
!
Fallback — Manual repos.conf entry

If eselect fails or GitHub prompts for credentials, create the entry manually:

/etc/portage/repos.conf/cachyos.conf 
[CachyOS-kernels]
location = /var/db/repos/CachyOS-kernels
sync-type = git
sync-uri = https://github.com/Szowisz/CachyOS-kernels.git
auto-sync = yes
bash 
emaint sync -r CachyOS-kernels
ℹ If git still prompts for GitHub credentials, disable the credential helper: git config --global credential.helper ""
// keyword unmasking
3
Accept ~amd64 keywords

Note: package.accept_keywords is a directory on most installs. Write to a file inside it:

bash 
echo "sys-kernel/cachyos-kernel ~amd64" >> /etc/portage/package.accept_keywords/cachyos
echo "sys-kernel/cachyos-sources ~amd64" >> /etc/portage/package.accept_keywords/cachyos
echo "virtual/dist-kernel ~amd64" >> /etc/portage/package.accept_keywords/cachyos

Verify:

bash 
cat /etc/portage/package.accept_keywords/cachyos

Expected output:

expected output
sys-kernel/cachyos-kernel ~amd64
sys-kernel/cachyos-sources ~amd64
virtual/dist-kernel ~amd64
⚠ Do not use >> to append to /etc/portage/package.accept_keywords directly — it's a directory and will throw an error. Always target a file inside it.
// installation
4
Install the CachyOS kernel

If emerge pulls in more masked packages, use --autounmask first to let portage write all needed keyword changes automatically:

bash — optional autounmask 
emerge --ask --autounmask sys-kernel/cachyos-kernel

Then apply the changes portage suggests and install:

bash 
emerge --ask sys-kernel/cachyos-kernel

This will automatically:

what happens
# 1. Install the prebuilt CachyOS kernel (BORE/SCHED-EXT, LLVM/LTO)
# 2. Invoke installkernel
# 3. dracut generates the initramfs
# 4. EFI stub places kernel + initramfs onto your ESP
# 5. uefi-mkconfig creates a UEFI boot entry
// verification
5
Verify EFI boot entry and kernel on ESP

Check that an EFI entry was created automatically:

bash 
efibootmgr -v

Check the kernel was placed on the ESP and that the cmdline is present in the entry:

bash 
ls /boot/EFI/Gentoo/
# Should show vmlinuz-*.efi

efibootmgr -v | grep -i gentoo
# Confirm root=UUID= appears in the entry
⚠ If the output shows "Warning! Kernel command root= is missing", your cmdline was not picked up. Double-check /etc/dracut.conf.d/00-installkernel.conf and re-run uefi-mkconfig.

If no entry was created, add one manually:

bash — manual EFI entry 
efibootmgr --create \
  --disk /dev/nvme0n1 \     # your disk
  --part 1 \                # your ESP partition number
  --label "CachyOS" \
  --loader /EFI/Gentoo/vmlinuz-7.0.0-cachyos-cachyos-dist.efi
6
Reboot and confirm
bash 
reboot

After booting, verify the running kernel:

bash 
uname -r
# Expected: something like 7.0.0-cachyos
